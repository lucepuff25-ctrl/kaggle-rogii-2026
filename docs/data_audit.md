# ROGII 真实数据审计

## 核心字段

| 文件 | 字段 | 定义/用途 |
|---|---|---|
| train horizontal | `MD,X,Y,Z` | 测深与逐点坐标 |
| train horizontal | `ANCC,ASTNU,ASTNL,EGFDU,EGFDL,BUDA` | 仅训练集提供的地层顶深字段 |
| train horizontal | `GR` | 水平井伽马测井值 |
| train horizontal | `TVT` | 回归标签；每行真实地质深度/厚度坐标 |
| train horizontal | `TVT_input` | 预测起点前等于 `TVT`，起点后为空 |
| test horizontal | `MD,X,Y,Z,GR,TVT_input` | 不含 `TVT` 和六个地层顶深字段 |
| typewell | `TVT,GR` | 配对垂直井曲线 |
| train typewell | `Geology` | 地层名称；测试 typewell 不提供 |

所有数值列均未发现无穷值。井 ID 来自 8 位文件名前缀；训练 773 个、测试 3 个，集合内均唯一。

主要观测范围：训练 `MD` 9,335–22,964、`X` 2,855,426.45–3,039,107.26、`Y` 1,007,175.50–1,141,857.53、`Z` -11,027.65–-7,174.22、`GR` 13.88–487.03。测试仅三井，范围明显更窄，且测试 `GR` 最大值 219.81；这属于分布覆盖差异，不能据此删除训练极值。

## 标签与目标分布

预测目标为水平井 `TVT`，提交列名为小写 `tvt`。训练全部 5,092,255 个标签均有限；均值 11,503.644、标准差 639.971、范围 9,245.19–12,893.89。基于固定 100,000 行蓄水池样本的分位数估计：P01 10,082.80、P25 10,989.43、P50 11,354.05、P75 12,036.94、P99 12,801.89。

训练有 3,783,989 个预测区行（`TVT_input` 为空）；已知区 1,308,266 行的 `TVT_input` 全部等于 `TVT`。测试预测区共 14,151 行。

## MSE/RMSE官方来源冲突

- Kaggle官方API竞赛元数据返回`evaluationMetric = Mean Squared Error`。
- Kaggle官方Evaluation页面和随附PPTX第14页均写Root Mean Squared Error（RMSE）。
- `MSE = (1/n) × Σ(y_i - ŷ_i)²`。
- `RMSE = sqrt(MSE)`。
- 两者对同一组预测通常产生单调一致的排名，但记录的分数数值和量纲不同。
- 在组织方统一说明前，本地代码同时提供MSE和RMSE，后续实验必须同时记录`cv_mse`和`cv_rmse`，不得把任一名称写成唯一已确认指标。

## 缺失、异常与重复

- 训练水平井：`GR` 缺失 1,507,972；`TVT_input` 缺失 3,783,989；`ANCC` 缺失 45,634；`EGFDL` 缺失 6,067。
- 测试水平井：`GR` 缺失 3,784；`TVT_input` 缺失 14,151。
- 训练 typewell：`Geology` 缺失 523,474；`TVT/GR` 无缺失。
- 全部 CSV 文件内未发现完全重复行，也未发现无穷值。
- 数值float64规范化后仍有752个唯一typewell轮廓；13组被多个井完全复用，共涉及34个井，最大一组10个井。这是同源近重复和CV泄漏风险。

## 训练/测试差异与直接重叠

公开测试样例井 `000d7d20`、`00bbac68`、`00e12e8b` 也出现在训练目录。逐行比较共同输入字段 `MD,X,Y,Z,GR,TVT_input`：三井 19,221/19,221 行全部完全一致，训练副本另含真实 `TVT`。因此训练目录暴露了公开样例的 14,151 个目标；这不证明Kaggle隐藏评分数据同样泄漏。任何诚实实验仍必须隔离这三井，禁止用其训练标签、目标统计或派生特征。

训练还比测试多六个地层顶深字段；这些字段只可作为训练标签侧信息，不能直接作为测试可用特征。

## sample_submission

- 列和顺序严格为 `id,tvt`，共 14,151 行。
- `id` 格式为 `{well_id}_{零基行号}`，全部唯一、合法且未越界。
- 三井分别 3,836、6,014、4,301 行。
- 每个 ID 恰好对应测试水平井中 `TVT_input` 为空的一行；无缺漏或多余 ID。
- 评分按行位置消费预测；生成提交时必须按 sample submission 左连接，不能依赖文件遍历或排序后的隐式顺序。

## 泄漏风险

1. **公开样例标签重叠：**公开测试三井在训练目录存在完整 `TVT`；隐藏评分数据状态未知。
2. **井内序列泄漏：**随机拆行会把同一井相邻深度点分到训练和验证。
3. **typewell 同源泄漏：**34 个井复用完全相同的 typewell 曲线。
4. **训练专属字段：**六个地层顶深列和 `Geology` 不可作为测试时可用输入。
5. **预测边界泄漏：**预测起点后的 `TVT` 只可作标签；所有预处理和派生统计须在 fold 内拟合。

## 尚未确认

- 数据无官方“区域”字段；`X/Y` 是坐标，但没有官方区域边界或空间块尺度，不能把任意网格解释为区域。
- Kaggle官方API写MSE，而官方Evaluation页面和随附PPTX写RMSE；该冲突仍待组织方统一说明。
- 未定义数值容差意义下的“近重复”；本次已检查完全重复行、完全相同 typewell 轮廓及训练/测试逐行重叠。
